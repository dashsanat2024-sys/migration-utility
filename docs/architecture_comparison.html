Stage 1 — Deterministic profiler (no LLM cost)
Type inference (boolean/date/enum/identifier/free_text) Format detection (ISO date, UK postcode, etc.)

Stage 3 — Human approval (nothing auto-commits)
Approve / Edit / Ask AI to reconsider ai_suggested vs human_confirmed flag per rule

  • Target/CMP sample extract with 15-column data dictionary
  • Column profiler auto-runs on upload (boolean/date/enum/id detection)

  • Deterministic column profiler (15 columns, 6 type categories)

  • Null rate, distinct value, format-pattern detection
Candidate Selection Module NOT BUILT Switchable Selection UI, Criteria Rules, Criteria Logic Not implemented. Roadmap item. All records in an extract are currently treated as candidates.
Migration Run Module PARTIAL Migration Run UI, Exclusion/Inclusion UI, Re-Runs UI Sync run ✓. Re-Run via worker resume (partial).
  • Async worker (same Docker image, separate deployment, queued)
  • Run progress tracking with resume on failure
  • Exclusion/Inclusion UI: navigation item exists, screen not fully built
Data Mapping Module MAJOR + Data Mapping Matrix UI, Tariff Mapping UI, Mapping Agreement Workflow Full mapping screen ✓. Tariff mapping partial.
  • Destination-as-plugin architecture (live schema contract, not manual upload)
  • Kraken GraphQL schema introspected live (AccountType, 80+ real fields)
  • Required/optional from GraphQL non-null markers directly
  • AI semantic field matching (LangChain structured output, MappingSuggestion)
  • Enum gap detection — source values with no destination match flagged not guessed
  • Workflow enhanced: draft → in_review → approved → signed_off (was implied 2-state)
Validation & Transformation Module MAJOR + Validation Rules UI, Transformation Rules UI, Validation Logic Fn, Transformation Logic Fn All 4 original capabilities built ✓.
  • AI-inferred transform rules from sample data (2-stage pipeline: profiler + LLM)
  • Confidence score 0.0–1.0 per rule; below 0.75 surfaced for review first
  • TransformRule Pydantic schema with lookup_table, uncovered_source_values, reasoning
  • Human approval queue (ai_suggested vs human_confirmed flag)
  • Transform types: copy, lookup, concat, conditional, date_reformat, pad_left, constant, regex
Migration Engine Datastore ENHANCED Rules & Configs DB, Candidate Statuses & History DB, Errors & Rejections DB All 3 original tables ✓ (PostgreSQL 16, Alembic migrations).
  • Full audit log: actor, role, comment, timestamp per workflow state transition
  • AI suggestion history stored separately from confirmed rules
  • Alembic versioned migrations for schema evolution
Migration Reporting Module ENHANCED Standard Reconciliation & Progress Reports, Standard Error Reports, Custom Reports Standard reports ✓. Custom report builder not built.
  • LangGraph multi-step AI error triage (fetch → cluster → explain → report)
  • 920 Kraken error codes catalogued (KT-CT-* core + KT-GB-* UK-specific)
  • Plain-English failure narration per error cluster
  • Reconciliation funnel (staged → selected → loaded → failed)
  • Exception queue with assign/override/resolve and audit per exception
  • BI export (JSON for Metabase, Power BI, or any dashboard)
API Connectors → Kraken ENHANCED Product Import Fn, Account Import Fn, Post Account Import Fn, Document Transfer Fn, DD Mandate Fn All 5 Kraken connectors implemented ✓.

  • SAP CRM plugin ✓ (documented). File export plugin ✓.
